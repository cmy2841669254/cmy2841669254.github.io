# 开始

Vue.js 最标志性的功能就是其低侵入性的响应式系统。组件状态都是由响应式的 JavaScript 对象组成的。当更改它们时，视图会随即自动更新。这让状态管理更加简单直观，但理解它是如何工作的也是很重要的，这可以帮助我们避免一些常见的陷阱。

我们先来创建一个最简易的 Vue3 应用（cdn的形式，无需构建工具）：

<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>VUE3 APP</title>
  <link rel="icon" href="https://vuejs.org/logo.svg">
</head>
<body>
  <div id="app"></div>
  <script type="module">
    import {
      createApp,
      ref,
      reactive
    } from 'https://unpkg.com/vue@3/dist/vue.esm-browser.js';

    const app = createApp({
      name: 'App',
      setup() {
        const count = ref(1);
        const state = reactive({ name: '字段名1' });
        const clickEvent1 = () => {
          count.value = count.value + 1;
        };
        const clickEvent2 = () => {
          state.name = state.name + 1;
        };

        return {
          count,
          state,
          clickEvent
        };
      },
      template: `
        <div>{{ count }}</div>
        <div>{{ state.name }}</div>
        <div>
          <button style="margin-right: 12px;" @click="clickEvent1">按钮1</button>
          <button @click="clickEvent2">按钮2</button>
        </div>
      `
    });
    app.mount('#app');
  </script>
</body>
</html>

界面展示：

1
字段名1
我们通过引入vue3的esm cdn文件，创建了一个vue3应用，引入ref、reactive这两个api，创建了两个响应式数据对象count和state。每当我们点击按钮更改字段count.value的值或是state.name的值时，视图都会进行更新。

# 使用 ref 和 reactive 创建响应式数据

ref 和 reactive 都可以创建一个响应式数据。不过 reactive api是有限制的，传入的参数仅对对象类型有效，比如对象、数组、Map等；对string、number等这类原始数据类型则无效。

const reactiveObj = reactive({}); // ok
const reactiveStr = reactive('str'); // 并不会创建响应式数据
/* 上一行代码中，浏览器控制台会警告字符串不可以被响应式化，当你使用ts时会告诉你参数类型错误 */
/* 相当于 const reactiveStr = 'str'; */

ref 则没有这些限制，ref 传入的参数可以是对象类型也可以是string、number这类的原始数据类型。

const objRef = ref({}); // ok
const strRef = ref('str'); // ok

尽管 ref 没有数据类型的限制，但是在读取值和设置值上比较繁琐：

/* 声明两个响应式数据对象，objRef 和 reactiveObj */
const objRef = ref({ count: 0 });
const reactiveObj = reactive({ count: 0 });

// objRef读取属性count的值
console.log(objRef.value.count); // 0
// reactiveObj读取属性count的值
console.log(reactiveObj.count); // 0

// objRef设置属性count的值
objRef.value.count = 1;
// reactiveObj设置属性count的值
reactiveObj.count = 1;

可以看到 ref 想要读取、设置某个属性值时必须要经过 value 属性，而 reactive 就跟对象那样读取和设置属性值。

在我们创建的Vue3应用中，我们能够知晓的是使用ref、reactive创建的响应式数据对象的属性值发生了改变，视图也会随之改变：

const state = reactive({ name: '字段名1' });

// 1秒后更改字段state.name
setTimeout(() => {
  state.name = '字段名2'; // 界面进行更新
}, 1000);

现在，我们观察以下代码：

const reactiveData = { welcome: 'Hello Vue.js.' };
const render = () => {
  const body = document.body;
  body.innerHTML = `<div>${reactiveData.welcome}</div>`;
};

render();
setTimeout(() => {
  reactiveData.welcome = 'Hello My Friends.'; // 值改变后，如何将render函数重新执行？
}, 1000);

我们声明了一个对象reactiveData，一个函数render；函数render内部是渲染body标签里的内容，内容里与对象reactiveData的welcome属性有关；调用函数render的1秒后，将对象reactiveData的welcome属性进行更改，我们希望字段reactiveData.welcome改变后函数render能够重新执行。怎么做能够实现值改变，render函数重新执行呢？

（扩展）在深入响应式原理前，我们先将上述代码更改一下：

setup() {
  let count = ref(1);
  let state = reactive({ name: '字段名1' });

  setTimeout(() => {
    count = ref(2);
    state = reactive({ name: '字段名2' });
  }, 1000);

  return { count, state };
},
template: `
  <div>{{ count }}</div>
  <div>{{ state.name }}</div>
`

我们在setup中加入了定时器，在一秒后，将变量count和state重新赋值，这个时候视图会进行更新吗？

# getter 和 setter

从ES5开始，对象的属性除了直接定义外，还可以通过修改"访问器描述符"中的getter和setter函数来实现对【对象】某个键的值读取、设置行为中执行额外的操作，这就是你常常听到的"vue2数据劫持"。我们来看一下ES5提供的一个api，Object.defineProperty()：

Object.defineProperty() 静态方法会直接在一个对象上定义一个新属性，或修改其现有属性，并返回此对象。

/* 语法 */
Object.defineProperty(obj, prop, descriptor)
// 参数obj：要定义属性的对象
// prop：一个字符串或 Symbol，指定了要定义或修改的属性键
// descriptor：要定义或修改的属性的描述符

对象中存在的属性描述符有两种主要类型：【数据描述符】和【访问器描述符】。【数据描述符】是一个具有可写或不可写值得属性。访问器描述符是由getter/setter函数对描述得属性。描述符只能是这两种类型之一，不能同时为两者。

数据描述符我们暂不去了解，我们先来看看访问器描述符中的getter和setter函数：

const obj = {};
let value = 1; // 对象obj的value属性值
let count = 0; // 读取对象obj的value属性次数

/* 为对象obj定义value属性 */
Object.defineProperty(obj, 'value', {
  /**
   * 属性是否可以被配置，通俗易懂的来说就是属性是否可以被删除
   * 删除后设置的属性描述符将不会生效，需要重新定义
   **/
  configurable: true,
  /**
   * 属性是否可以出现在obj的属性枚举中
   * 通俗易懂的来说就是属性是否可以被for...in、Object.keys()方法遍历出来
   **/
  enumerable: true,
  /* 用作属性 getter 的函数 */
  get() {
    count++;
    console.log(`value属性被读取了${count}次`);
    return value;
  },
  /* 用作属性 setter 的函数 */
  set(newVal) {
    console.log('value属性被重新设置');
    value = newVal;
  }
});

console.log(obj.value); // 触发getter函数
obj.value = 2; // 触发setter函数

上述代码中，我们为对象obj定义了value属性，为value属性重新设置了getter和setter函数，每当我们读取obj.value时，触发getter函数的执行，在getter函数中，记录下属性value会被读取的次数并返回值；每次为obj.value重新赋值时，触发setter函数执行，setter函数中打印"value属性被重新设置"。

回到我们提出疑问的代码里，我们用Object.defineProperty()来改造下：

let welcome = 'Hello Vue.js.';
const reactiveData = {};

const render = () => {
  const body = document.body;
  body.innerHTML = `<div>${reactiveData.welcome}</div>`;
};

Object.defineProperty(reactiveData, 'welcome', {
  configurable: true,
  enumerable: true,
  get() {
    return welcome;
  },
  set(newVal) {
    welcome = newVal;
    render(); // 执行render函数
  }
});

render();
setTimeout(() => {
  reactiveData.welcome = 'Hello My Friends.'; // 更改reactiveData.welcome的值，触发setter函数执行
}, 1000);

我们每次更改对象reactiveData.welcome时都会重新对重新执行render函数，现在看起来，有那么一点vue的既视感，数据变更，视图也会变化。但是Object.defineProperty拿来创建"响应式数据"是有局限性的：

示例代码如下：

let welcome = 'Hello Vue.js.';
const reactiveData = {};

const render = () => {
  const body = document.body;
  body.innerHTML = `<div>${reactiveData.welcome}</div>`;
};

Object.defineProperty(reactiveData, 'welcome', {
  configurable: true,
  enumerable: true,
  get() {
    return welcome;
  },
  set(newVal) {
    welcome = newVal;
    render();
  }
});

render();
setTimeout(() => {
  /* 删除了对象reactiveData的welcome属性，界面仍然是"Hello Vue.js." */
  delete reactiveData.welcome;
}, 1000);

上述代码中，我们在界面渲染后，等待1秒钟，删除了对象reactiveData的welcome属性，界面没有变化。因为我们Object.defineProperty()只定义了getter和setter，只有触发setter才会调用render函数更新界面。而且Object.defineProperty()方法中并没有提供对应删除属性值的描述符，这意味着需要你手动重新渲染界面：

// 省略其他代码
setTimeout(() => {
  delete reactiveData.welcome;
  render(); // 手动重新执行render函数
}, 1000);

虽然从界面的反馈上可以看出代码片段 [delete reactiveData.welcome]等价于[reactiveData.welcome = undefined]，而且[reactiveData.welcome = undefined]能触发setter函数的执行，重新执行render函数渲染界面；不过删除属性值和赋值为undefined毕竟是两种行为，那么有什么办法能够知晓属性被删除呢？

# Proxy（代理）

Proxy 对象用于创建一个对象的代理，从而实现基本操作的拦截和自定义（如属性查找、赋值、枚举、函数调用等）。我们先来熟悉下语法：

/* 语法 */
const p = new Proxy(target, handler);
// target：要使用 Proxy 包装的目标对象（可以是任何类型的对象，包括原生数组、函数，甚至另一个代理）
// handler：一个通常以函数作为属性的对象，各属性中的函数分别定义了在执行各种操作时代理 p 的行为

我们先来创建一个简单的示例，快速上手Proxy：

/* 原始数据 */
const rowData = { value: 1 };

/* 代理原始数据的代理对象 */
const proxyData = new Proxy(rowData, {
  /* 捕获读取属性的操作 */
  get(target, property, receiver) {
    /* get函数总共有3个参数 */
    // target：传给Proxy的第一个参数，即本示例中的对象rowData
    // property：读取的属性值
    // receiver：暂时不管
    console.log(`捕获到读取${property}属性操作`);
    return target[property]; // 将值返回出去
  },
  /* 捕获设置属性操作 */
  set(target, property, value, receiver) {
    /* set函数总共有4个 */
    // target：传给Proxy的第一个参数，即本示例中的对象rowData
    // property：被设置值的属性
    // value：设置的值
    // receiver：暂时不管

    /* 注意！set方法应当返回一个布尔值 */
    // 返回true代表属性设置成功，不过就算你不返回也没什么事
    console.log(`捕获到设置${property}属性操作`);
    target[property] = value;
    return true;
  },
  /* 捕获delete操作 */
  deleteProperty(target, property) {
    /* deleteProperty函数只有两个参数 */
    // 参数的含义与以上注释一致

    /* 注意！deleteProperty函数必须返回一个布尔值，表示该属性是否被成功删除 */
    console.log(`捕获到删除${property}属性操作`);
    delete target[property];
    return true;
  }
});

console.log(proxyData.value);
proxyData.value = 3;
console.log(rowData);
setTimeout(() => {
  delete proxyData.value;
}, 1000);

以上代码中，我们重新定义了捕获属性的读取、设置和删除操作函数，当然 Proxy 的捕获器函数不止这些，后续我们会展示 Proxy 其他的捕获器函数，现在我们使用 Proxy 来代替下 Object.defineProperty() 重新实现下：

const rowData = { welcome: 'Hello Vue.js.' };
const render = () => {
  const body = document.body;
  body.innerHTML = `<div>${reactiveData.welcome}</div>`;
};
const reactiveData = new Proxy(rowData, {
  get(target, prop) {
    return target[prop];
  },
  set(target, prop, value) {
    target[prop] = value;
    render();
    return true;
  },
  deleteProperty(target, prop) {
    delete target[prop];
    render();
    return true;
  }
});

render();
setTimeout(() => {
  reactiveData.welcome = 'Hello My Friends.';
}, 1000);
setTimeout(() => {
  delete reactiveData.welcome;
}, 2000);

可以看到 Proxy 相较于 Object.defineProperty() 能额外捕获到像删除属性这样的操作，目前我们的示例都还很简单，相信你看出来以上的示例其实是有问题的，render 函数在捕获函数 set、deleteProperty 中写死的，意味着如果修改或删除了与视图无关的属性，也会重新渲染，这显然是不对的：

const rowData = { welcome: 'Hello Vue.js.', flag: true };
const render = () => {
  console.log('开始渲染！');
  const body = document.body;
  body.innerHTML = `<div>${reactiveData.welcome}</div>`;
};
const reactiveData = new Proxy(rowData, {
  get(target, prop) {
    return target[prop];
  },
  set(target, prop, value) {
    target[prop] = value;
    render();
    return true;
  }
});

render();
setTimeout(() => {
  reactiveData.welcome = 'Hello My Friends.';
}, 1000);
setTimeout(() => {
  console.log('即将修改与视图无关的属性');
  reactiveData.flag = false;
}, 2000);

上述代码展现了在捕获器函数中写死 render 函数导致的问题，render 函数中只跟代理对象 reactiveData 的属性 welcome 有关，跟属性 flag 是一点关系都没有，flag属性的重新设置理应不能导致render函数的重新执行；现在问题的所在便是我们没有做到视图与响应式数据之间依赖关系--"依赖收集"。

# 总结

Proxy的捕获器get函数与set函数与Object.defineProperty()访问器描述符的getter和setter相似，而且Proxy捕获器能感知更多的操作对象的行为（删除属性、in操作符劫持、函数调用劫持、new操作符劫持等），从创建响应式数据上看，Proxy简直是强化版的getter和setter。

我们回想下vue2在创建响应式数据的一些细节：

<script>
export default {
  data() {
    return {
      state: {}
    };
  },
  methods: {
    clickEvent() {
      this.state.name = '你好'; // 无法触发页面的更新
      /* 如果想触发页面的更新则需要重新设置this.state */
      /*
      this.state = { name: '你好' }; // 触发页面的更新
      */
    }
  }
};
</script>

<template>
  <div>
    <h1>{{ state.name }}</h1>

    <div>
      <button @click="clickEvent">按钮</button>
    </div>
  </div>
</template>

vue3则没有上述的问题：

<script setup>
import { reactive } from 'vue';

const state = reactive({});

const clickEvent = () => {
  state.name = '你好'; // 直接触发页面的更新
};
</script>

<template>
  <div>
    <h1>{{ state.name }}</h1>

    <div>
      <button @click="clickEvent">按钮</button>
    </div>
  </div>
</template>

不过我们换一个角度去看，getter和setter只关注单个属性的读取和设置操作，而Proxy的捕获器函数get和set，读取、设置任意属性都会触发对应捕获器函数。上文中提到的ref创建的响应式数据，读取、设置值都经过value属性，我们是不是可以猜测下ref的实现与getter/setter有关？看了以上vue2和vue3的对比，我想你大概也知道了reactive方法内部是采用了哪个api来创建响应式数据了。
